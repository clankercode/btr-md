# Filesystem containment lives in path_scope; URL/link semantics live in pmd-core

Path/URL helper functions (normalize, canonicalise, is_within, scheme checks, markdown-extension checks, reference-label scanning) were independently reimplemented across five security-relevant pmd-app modules. We decided deduplication has exactly two homes: filesystem containment primitives (canonicalise, is_within, path normalization) concentrate in `path_scope`; link/URL *semantics* (classification, scheme parsing) belong to pmd-core's facts interface, which already computes them. We explicitly rejected a single app-side "policy" module absorbing both — a grab-bag layer would blur the seam between filesystem safety and document semantics and re-shadow classifications pmd-core already owns (an earlier duplicate, `classify_target`, survived only on a test path for exactly this reason).
